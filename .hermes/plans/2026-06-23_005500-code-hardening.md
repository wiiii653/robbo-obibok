### Task 1: Replace `id(source)` with monotonic counter

**Objective:** Eliminate theoretical `id()` reuse race in `_after_stream_end`.

**Files:**
- Modify: `robbo-obibok.py:238-249` (`_after_stream_end`)
- Modify: `robbo-obibok.py:934,992,1147,1217,1272,1391,2906` (7 lambda callers)

**Change:** Replace `id(source)` in lambdas with `_next_source_id` counter.

```python
# Module-level (near active_streams dict, ~line 237)
_next_source_id: int = 0

def _after_stream_end(guild_id, error, source_id=0):
    # ... existing logic using source_id remains unchanged
```

In each lambda, replace `sid=id(source)` with `sid` captured from counter:

```python
# Before (7 places):
source = MonitorAudioSource(SINK_NAME)
state.vc.play(
    source,
    after=lambda e, sid=id(source): _after_stream_end(state.guild_id, e, sid),
)
active_streams[state.guild_id] = source

# After:
global _next_source_id
_next_source_id += 1
sid = _next_source_id
source = MonitorAudioSource(SINK_NAME)
state.vc.play(
    source,
    after=lambda e, sid=sid: _after_stream_end(state.guild_id, e, sid),
)
active_streams[state.guild_id] = source
```

**Verification:** Counter is monotonic → never repeats. Old stale callbacks with stale `sid` values never match.

**Risk:** Near zero. Counter wraps after 2^63 in Python bigints (never in practice).

### Task 2: Fix signal handler thread safety

**Objective:** Replace unsafe `loop.create_task()` in signal handler with `asyncio.run_coroutine_threadsafe()`.

**Files:**
- Modify: `robbo-obibok.py:3750-3756` (`handle_signal`)

**Change:**

```python
def handle_signal(signum, frame):
    """Signal handler: schedule shutdown gracefully. Thread-safe."""
    log.info("Received signal %d, shutting down...", signum)
    loop = asyncio.get_event_loop()
    if loop.is_running():
        asyncio.run_coroutine_threadsafe(graceful_shutdown(), loop)
        loop.call_soon_threadsafe(lambda: asyncio.ensure_future(bot.close()))
```

**What changes:** `loop.create_task(graceful_shutdown())` → `asyncio.run_coroutine_threadsafe(graceful_shutdown(), loop)`.

**Verification:** On `kill -TERM` or `Ctrl+C`, bot shuts down gracefully. No deadlock risk.

**Risk:** Low. `run_coroutine_threadsafe` is the documented way to schedule coroutines from non-async threads.

### Task 3: Limit `cleanup_orphan_players` to current user

**Objective:** Prevent `pkill -x audacious` from killing other users' Audacious processes.

**Files:**
- Modify: `robbo-obibok.py:3003-3006` (`cleanup_orphan_players`)

**Change:**

```python
def cleanup_orphan_players():
    """Kill orphaned audacious processes from crashed bot sessions.
    Limited to current user to avoid killing system-wide processes."""
    subprocess.run(["pkill", "-u", os.environ.get("USER", ""), "-x", "audacious"], capture_output=True)
```

**Verification:** `ps aux | grep audacious` before/after — only current user's processes are killed.

**Risk:** Near zero. `-u` flag is standard POSIX.

### Task 5: Atomic JSON writes

**Objective:** Prevent JSON corruption on crash/power-loss during file save.

**Files:**
- Modify: `robbo-obobok.py:2136-2142` (`save_favorites`)
- Modify: `robbo-obobok.py:2244` (`save_blacklist`)
- Modify: `robbo-obobok.py:288-304` (`save_queue`)
- Modify: `robbo-obobok.py:2151-2170` (`save_playlist`)

**Change:** Write to `.tmp` then `os.rename()` (atomic on same filesystem).

```python
def _atomic_json_write(path: str, data: dict) -> None:
    """Write JSON atomically: tmp → rename to prevent partial writes."""
    tmp_path = path + ".tmp"
    try:
        with open(tmp_path, "w") as f:
            json.dump(data, f, indent=2)
        os.replace(tmp_path, path)  # atomic rename (POSIX)
    except Exception as e:
        log.error("Failed to save %s: %s", path, e)
        try:
            os.unlink(tmp_path)
        except OSError:
            pass

def save_favorites(data: dict):
    _atomic_json_write(FAVORITES_FILE, data)
```

Apply the helper to all 4 save functions.

**Verification:** Kill bot with `SIGKILL` mid-write → file is either intact (old version) or the new `.tmp` file. No partial JSON. On restart, reads fine.

**Risk:** Near zero. `os.replace()` is POSIX `rename(2)` — atomic on same filesystem.
